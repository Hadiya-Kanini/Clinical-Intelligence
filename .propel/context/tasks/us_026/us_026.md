## Story ID
   * ID Format: US_026

## Story Title
   * Send password reset emails via SMTP

## Description
  * As a user, I want to receive a password reset email with a secure link when I request a reset, so that I can complete the password recovery process.

## Acceptance Criteria
  * Given a valid reset request, When processed, Then an email with the reset link is sent via SMTP.
  * Given the reset email, When sent, Then it uses TLS 1.2+ for secure transmission.
  * Given the reset link, When included in email, Then it contains the secure token and points to the reset page.
  * Given email sending, When attempted, Then delivery status is logged for troubleshooting.

## Edge Cases
   * What happens when email delivery fails (SMTP error)?
   * What happens when the user's email inbox is full?
   * How does the system handle email rate limiting from the SMTP provider?
