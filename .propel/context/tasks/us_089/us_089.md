## Story ID
   * ID Format: US_089

## Story Title
   * Display aggregated patient profile and clinical content

## Description
  * As a clinician, I want to see aggregated patient profile and clinical content in the Patient 360 Dashboard, so that I have a complete view of the patient.

## Acceptance Criteria
  * Given Patient 360, When displayed, Then Patient Profile shows demographics, document info, allergies, vitals, social history (FR-072).
  * Given Patient 360, When displayed, Then Clinical Content shows problems, medications, procedures, lab results, plan (FR-073).
  * Given the dashboard, When rendered, Then data is aggregated from all processed documents.
  * Given the Standard User Patient 360 page, When accessed, Then it displays the complete view (FR-120).

## Edge Cases
   * What happens when some categories have no data?
   * How does the system handle conflicting data that hasn't been resolved?
   * What happens when documents are still processing?
